2. 학교 강의내용 공부
	<데이터통신>
	- ISO에서 OSI 7계층 제안 => 하지만 TCP/IP를 대체하진 못함.
		1) 이미 TCP/IP 모델이 자리잡음(바꾸려면 비용이 든다.)
		2) OSI 일부 계층이 완전히 정의되지 않음(세션, 표현계층 애매함)
		3) TCP/IP 보다 충분히 높은 성능을 발휘하지 못함.
		=> 참조 모델로만 남게 됨.

	- OSI 7계층
		물리 - 데이터링크 - 네트워크 - 전송 - 세션 - 표현 - 응용 계층

	- TCP/IP 5계층
		물리 - 데이터링크 - 인터넷 - 전송 - 응용 계층
	-> 각 계층에는 해당 계층끼리 연결할 수 있는 프로토콜 존재

	- 전송 데이터의 캡슐화와 역캡슐화
		> 캡슐화 : 헤더 혹은 트레일러(2계층에서 추가, 오류제어 역할)를 더하여 데이터 전송
		> 역캡슐화 : 각 계층에 맞는 헤더 제함, 최종 목적지에 원 데이터가 수신됨.
		> 데이터 단위
			1) SDU(서비스 데이터 단위)
			-> 상, 하위 계층 간 인터페이스로 전달되는 실제 정보
			2) PDU(프로토콜 데이터 단위)
			-> 전달받은 SDU에 헤더(PCI)를 추가한 데이터

	- 계층별 기능
		> 물리 계층(물리적 연결)
		> 데이터링크 계층(전송받은 데이터에 헤더와 트레일러를 덧붙임(프레임 구성, 캡슐화))
		> 네트워크 계층(IP 주소 이용, 종단 간 전달(시작과 끝에 전달), 단순한 데이터 전송)
		> 전송 계층(데이터에 의미 부여)
		> 세션 계층(전송 시 끊겼다가 다시 보내면 시작, 종료 지점을 파악하여(동기화) 처리함.)
		> 표현 계층(데이터 암호화, 압축, 데이터 표현 방식(ex : 파일 확장자))
		> 응용 계층(ex : www 등..... , 응용 프로세스가 네트워크에 접근할 수 있는 수단 마련)

	- OSI 각 계층에서 사용되는 데이터 단위(PDU)
		> 물리 계층(비트)
		> 데이터링크 계층(프레임)
		> 네트워크 계층(패킷, 데이터그램)
		> 전송 계층(세그먼트)
		> 세션, 표현, 응용 계층(메시지)

	
	- 정현파(주기적인 아날로그 신호)
		> 정현파는 진폭, 주파수, 위상으로 표현
		> s(t) = (진폭)sin(2파이 * (주파수) * t + (위상))

	- 대역폭(주파수 성분들의 범위)
		> 가장 높은 주파수 - 가장 낮은 주파수 = 대역폭

	- 유효 대역폭(신호의 대역폭 범위)

	- 채널 용량(전송 매체가 전송할 수 있는 최대 비트율)
		> 전송 매체의 대역폭(아날로그 : Hz, 디지털 : bps)
		> 신호 준위 수(부호화 방법)
		> 채널의 품질(잡음의 정도)

	- 무잡음 채널(나이퀴스트 비트율)
		> C(bps) = 2(대역폭)log2(신호 준위)

	- 잡음이 있는 채널(섀넌 용량)
		> C = (대역폭)log2(1 + (신호 대 잡음비))
		
	- 전송 손상
		> 감쇠(ex : 와이파이 공유기의 위치에 따른 신호 감쇠)
		> 왜곡(신호의 모양이나 형태가 변형되는 것.)
		> 잡음(원하지 않는 잡음 ex : 열잡음, 상호변조잡음, 누화, 충격 잡음 등....)

	
	- 브로드밴드
		> 광대역 전송, 데이터를 반송파에 실어서 보낸다.
		> 데이터를 반송파에 싣는 것(변조), 수신 측에서 데이터와 반송파를 구분하는 것.(복조) 필요

	- 베이스밴드
		> 기저대역 전송(근거리 전송)
		> 전송 데이터가 아날로그면 변조 필요, 아니면 데이터 그대로 전송

	
	- 링크(전송 매체)
		> 하나의 장치로부터 다른 장치로 데이터를 보내는 물리적인 통신 경로
		> 회선 연결 방식
		1) 점대점 회선 방식(두 장치를 전용 회선으로 직접 연결, 응답 속도가 빨라 고속 처리에 이용)
		2) 다중점 회선 방식(3개 이상의 장치가 하나의 링크 공유, 데이터 전송은 폴링(장치>메인), 셀렉션(메인>특정 장치)로 수행)
		3) 집선 회선 방식(단말 장치에서 저속으로 전송되는 데이터를 집선 장치에 모아서 메인 컴퓨터에 고속으로 전송)
		4) 다중 회선 방식

	- 직렬 전송
		> 한 번에 한 비트씩 순서대로 데이터 전송(ex : RS-232C, USB)
		> 원거리 전송에 적합, 1개의 데이터 전송회선 사용으로 비용 경제적
		> 직병렬 변환기 필요(복잡), 병렬 전송에 비해 전송 속도가 낮음.
		> 한 번 전송하는 것은 한 비트이지만 보낼 때 여러 비트를 전송하기 때문에, 병렬-직렬, 직렬-병렬 변환기가 필요.

	- 병렬 전송
		> 여러 개의 비트를 그룹으로 묶어 동시 전송
		> 필요 비트 수 만큼 전송로 필요 -> 전송 비용 높음, 잘 사용되지 않음.
		
	- 비동기식 전송(ex : 키보드)
		> 전송하는 문자의 구별 목적으로 시작과 끝을 알리는 시작, 종룔 비트 전송(보통 각 1비트씩의 크기)
		> 전송할 데이터가 없을 땐 휴지 상태(비트 1을 계속에서 전송)
		> 송수신 측이 항상 동기상태가 아님.

	- 동기식 전송
		> 데이터 블록 단위로 전송(문자 위주 방식, 비트 위주 방식)
		> 동기식 전송은 대용량 데이터 시 비동기 전송보다 효율적
		> 데이터 앞쪽에 반드시 동기 문자 포함(송수신 측의 동기화 목적)
		> 한 묶음 구성의 문자들 사이에는 휴지 간격이 없음.(비는 공간)
