2. 학교 공부 필기
	<알고리즘>
	
	- 이진 탐색 트리
		> 이진 트리를 탐색용 자료구조로 사용하기 위해 원소 크기에 따라 노드 위치를 정의한 것
		> 루트 값을 기준으로 루트 값보다 작으면 왼쪽, 크면 오른쪽에 위치한다.


	- 힙의 개념
		> 완전 이진 트리에 있는 노드 중 키값이 가장 큰 노드나 키값이 가장 작은 노드를 찾기 위해 만든 자료구조
		> 최대 힙(키값이 가장 큰 노드를 찾기 위한 완전 이진 트리)
		> 최소 힙(키값이 가장 작은 노드를 찾기 위한 완전 이진 트리)

	* 완전 이진 트리?
	-> 이진 트리 모양 중 값을 트리의 왼쪽부터 채우는 이진 트리를 말함


	- 이진 트리의 문제점
		> 트리의 높이가 깊어지면?(탐색 시간이 오래 걸림 -> O(n))
		> 균형이 맞지 않으면?(탐색 시간이 다름)
		> 힙(완전/균형 이진 트리) => 탐색 시간이 걸림, 높이가 n개일 때 2(의 n승) - 1 개의 원소 개수 존재


	- B 트리
		> 디스크의 접근 단위는 블록(페이지)
		> 디스크에 한 번 접근하는 시간은 수십 만 명령어의 처리 시간과 맞먹는다.
		> 검색트리가 디스크에 저장되어 있다면 트리의 높이를 최소화하는 것이 유리
		> B 트리는 다진검색트리가 균형을 유지하도록 하여 최악의 경우 디스크 접근 회수를 줄인 것

	<운영체제>

	- adduser 명령어(새로운 사용자 추가하는 명령어, /etc/group 파일에 새로운 행이 추가됨)

	- passwd 명령어(사용자의 비밀번호 변경)

	- chage 명령어(사용자의 비밀번호를 주기적으로 변경하도록 설정하는 명령어)

	- groupadd(새로운 그룹 생성), groupmod(그룹의 속성을 변경하는 명령어), groupdel(그룹 삭제), gpasswd(그룹 비밀번호 변경)


	- 링크 실습(우분투 리눅스)
		> mkdir link(link라는 디렉토리 생성)

		> cd link(link 디렉토리로 위치 이동)

		> vi basefile.txt(basefile이라는 텍스트 파일 생성)

		> basefile.txt 파일에 내용 넣기(아무거나 상관 없음)

		> :wq 로 저장 후 빠져나와 ls -l로 파일 생성 유무 확인

		> ln basefile.txt hardlink(하드링크 생성)

		> ls -il(파일을 확인해 보면 basefile.txt 파일과 hardlink의 아이디와 용량이 똑같은 것을 볼 수 있다.)

		> cat hardlink(basefile.txt 파일의 내용을 확인할 수 있음.)
		-> basefile.txt 파일의 내용을 수정하고 cat으로 hardlink 내용을 확인 해 보면 수정된 내용으로
		보여지는 것을 볼 수 있다.

		> ln -s basefile.txt softlink(softlink 생성)

		> ls -il로 확인 시 softlink는 파일 아이디도 다르고 파일의 용량도 basefile보다 적은 것을 볼 수 있다.
		(basefile 파일에 내용 추가 시 hardlink와 basefile은 용량이 늘어났지만 softlink는 용량이 그대로인 것을
		확인할 수 있다.)
		+ 추가로, cat 명령어로 softlink 내용을 출력하면 hardlink와 동일하게 파일의 수정된 내용이 반영되어
		보여지는 것을 확인할 수 있다.

		> 대신에, basefile의 위치를 mv로 이동한 후 hardlink와 softlink를 출력해 보면 hardlink는 내용이 그대로
		나오지만, softlink는 그런 파일이나 디렉토리가 없다고 나온다.
		(hardlink는 바뀐 위치에서의 basefile.txt의 내용을 수정해도 수정된 내용이 반영된다.)

		> 하드 링크 생성(ln 원본파일 링크파일명)
		> 소프트 링크 생성(ln -s 원본파일 링크파일명)


	- 프로세스(내용 관련 문제 나올 수 있음!)
		> 하드디스크에 저장된 실행 코드(프로그램)가 메모리에 로딩되어 활성화된 것
		> ex : 웹 브라우저 프로그램인 파이어폭스는 하드디스크의 어딘가에 저장되어 있는데 이렇게 하드디스크에
		저장된 파일을 "프로그램"이라 부름
		> 파이어폭스를 실행하여 화면에 나타난 상태를 "프로세스"라 부름
		> 포그라운드 프로세스, 백그라운드 프로세스로 나뉨.


	- 프로세스 번호
		> 각 프로세스에 할당된 고유 번호
		> 메모리에 로딩되어 활성화된 프로세스를 구분하려면 고유 번호가 필요
		> ps(현재 프로세스 상태 확인 명령어)
		> kill(프로세스 강제 종료 명령어)
		> pstree(부모/자식 프로세스 관계를 트리 형태로 보여줌)


	- 소켓
		> 소켓과 관련된 스크립트 파일은 /lib/systemd/system 디렉토리에 '소켓명.socket'으로 존재함(중요!)
		> 소켓이 언제 가동되는지 정도만 알아두기


	- 요약
		> 리눅스를 설치 시 기본적으로 root라는 이름의 슈퍼 유저가 있고, 이는 모든 작업을 실행할 수 있는 권한이 있음.
		> 사용자가 등록되어 있는 파일은 /etc/passwd, 그룹이 등록되어 있는 파일은 /etc/group이다.
		> 사용자 및 그룹 관리 명령어
			adduser(새로운 사용자 추가), passwd(사용자 비밀번호 변경), usemod(사용자 속성 변경)
			userdel(사용자 삭제), chage(사용자의 비밀번호를 주기적으로 변경하도록 설정)
			groups(사용자가 소속된 그룹 보여줌), groupadd(그룹 생성), groupmod(그룹 수정)
			groupdel(그룹 삭제), gpasswd(그룹의 비밀번호 설정)

		> touch 명령으로 생성한 파일의 정보(기억해 두기)
		-(파일 유형)rw-r--r--(파일 허가권) 1(링크 수) root(파일 소유자) root(파일 소유 그룹) 0(파일 크기) 날짜 파일명
	
		> 파일 유형은 파일의 종류를 나타냄, 디렉터리는 d, 일반적인 파일은 -로, 블록 디바이스는 b, 문자 디바이스는 c,
		링크는 1로 표시된다.
		
		> 허가권 표시(알아두기)
		rwx(읽기, 쓰기, 실행 권한)(4 2 1 크기)
		ex: 644(소유자 rw 권한, 그룹 r 권한, 그 외 사용자는 r 권한)

		> 파일 허가권을 변경하는 chmod 명령어는 root 사용자 또는 해당 파일의 소유자만 실행 가능

		> chgrp 명령어는 그룹만 변경 가능
		> 파일의 링크는 하드 링크, 소프트 링크로 구분됨.
		
		> 프로세스 관련 중요 개념
			포그라운드 프로세스(화면에 나타나서 사용자와 상호 작용하는 프로세스)
			백그라운드 프로세스(화면에 나타나지 않은 채 뒤에서 실행되는 프로세스)
			프로세스 번호(각 프로세스에 할당된 고유 번호)
			작업 번호(현재 실행 중인 백그라운드 프로세스의 순차 번호)
			부모/자식 프로세스(모든 프로세스는 부모 프로세스에 종속되어 실행됨)

		> 프로세스 관련 주요 명령어
			ps, kill, pstree

		> 데몬이라고도 부르는 서비스는 서버 프로세스를 말함.
		> 서비스는 평상시에도 늘 작동하는 서버 프로세스, 소켓은 필요할 때만 작동하는 서버 프로세스를 말함
		서비스와 소켓은 systemd라는 서비스 매니저 프로그램으로 작동하거나 관리함
		(데몬, 서비스의 차이점 기억해두기)


	- 문제 풀이(★)
	1) 사용자가 등록되어 있는 /etc/passwd 파일의 구성 내용 차례로 나열
	> 사용자 이름, 비밀번호, 사용자id, 사용자 소속 그룹 id, 추가 정보, 홈 디렉토리, 기본 셸

	2) 그룹의 정보가 등록되어 있는 파일은?
	> /etc/group

	3) 다음은 각각 어떤 명령어에 대한 설명인가?
	> 새로운 사용자를 추가하는 이 명령어를 실행하면 /etc/passwd, /etc/shadow, /etc/group 파일에 새로운 행 추가
	(adduser)
	> 사용자의 비밀번호를 주기적으로 변경하도록 설정
	(chage)
	> 사용자의 비밀번호 변경
	(passwd)
	> 그룹 속성 변경
	(groupmod)

	4) ls -l 명령을 실행했을 때 나타나는 정보가 아닌 것은?
		1. 파일 유형	2. 파일 이름	3. 파일 크기
		4. 최초 생성 일자	5. 파일 소유자	6. 링크 수
	(4번, 최초 생성 일자, 최초 생성 일자가 아닌 마지막 수정 날짜가 나옴.)

	5) ls -l 명령을 실행했을 때 나타나는 각 파일 유형을 의미하는 영문 한 자 또는 기호
	> 디렉토리 .d, 블록 디바이스 b, 링크 l, 문자 디바이스 c, 일반 파일 -

	6) 각 파일 허가권을 chmod 명령으로 수행할 때의 rwxrwxrwx 형식을 쓰시오.(시험은 이 방식으로 나올것)
	> 소유자는 읽거나 쓸 수 있고 그룹과 그 외 사용자는 읽을 수만 있다.(chmod 644 파일명)
	> 소유자와 그룹은 읽거나 실행할 수 있고 그 외 사용자는 실행만 가능(chmod 551 파일명)
	> 소유자만 읽을 수 있고 나머지는 아무것도 못함.(chmod 400 파일명)
	
	7) 각 파일 허가권을 chmod 명령으로 수행할 때의 ugo+rwx 또는 ugo-rwx 형식을 쓰시오.
	> 소유자에게 쓰기 권한을 준다(chmod u+w 파일명)
	> 그룹과 그 외 사용자의 쓰기 및 실행 권한을 제거한다(chmod go-wx 파일명)
	> 모든 사용자에게 읽기 권한 추가(chmod ugo+r 파일명)
	
	8) 다음 중 chown/chgrp 명령(이름 변경)에 대한 설명?
	chown : 파일이나 폴더 또는 하위경로의 소유자를 변경(파일, 폴더의 그룹, 사용자 변경할 수 있다.)
	chown [옵션] [소유자: 소유그룹] [파일 또는 폴더]
	chgrp : 파일의 사용자 그룹을 변경한다.
	chgrp [옵션] [그룹] [파일]

	9) 다음 중 링크에 대한 설명으로 옳지 않은 것은?
	> 심볼릭 링크는 원본 파일과 inode를 공유하지 않고, 원본 파일이 제거되도 하드 링크는 영향이 없음.

	10) 다음은 무엇에 대한 설명인지 보기에서 고르기
	1. 현재 실행 중인 백그라운드 프로세스의 순차 번호이다.(작업 번호)
	2. 파이어폭스와 마찬가지로 실행하면 화면에 나타나서 사용자와 상호 작용을 하는 프로세스이다.(포그라운드 프로세스)
	3. X윈도우가 종료되면 브라우저인 파이어폭스도 함께 종료된다.(부모 프로세스)
	보기 -> 포그라운드 프로세스, 백그라운드 프로세스, 프로세스 번호, 작업 번호, 부모 프로세스

	11) 다음은 각각 어떤 명령어에 대한 설명인가?
	1. 프로세스를 강제로 종료(kill)
	2. 부모 프로세스와 자식 프로세스의 관계를 트리 형태로 보여준다.(pstree)
	3. 현재 프로세스의 상태를 확인한다.(ps)

	12) 서비스를 시작, 중지, 재시작하는 명령어는 무엇인가?
	(systemctl)	

	13) 다음 빈칸에 알맞은 말을 넣으시오.
	(서비스) 는 평상시에도 늘 작동하는 서버 프로세스이고, (소켓) 는 필요할 때만 작동하는 서버 프로세스를 말한다.
