1. 생활코딩 문서 공부
	> WEB1 - HTML & Internet 내용 정리
		1) 어떠한 문제를 해결하기 위해 공부 시작(문제 해결을 위한 공부가 중점이 되는 것이 중요, 공부 자체가 문제가 되면 안 됨)

		2) 코딩 시작 전 전체적인 내용 기획 필요

		3) 원인(코드, 사람이 짜는 컴퓨터 언어) -> 결과(웹 혹은 앱 페이지, 코드를 바탕으로 기계가 결과를 냄)

		4) 웹 페이지 제작의 1단계 -> HTML, 웹은 퍼블릭 도메인(저작권 X)

		5) 코딩 전 에디터 선택 -> visual code, atom 등....

		6) 코딩 파일의 확장자는 .html

		7) 태그 관련 설명
		태그에는 기본적으로 여는 태그와 닫는 태그로 이뤄지며 사이에 내용이 들어감
		<태그>내용</태그>
		그러나 태그 중에는 <br>처럼 하나로만 이뤄진 태그들도 있음
		
		8) 배운 태그 종류들
			- <strong>내용</strong>	(내용 강조표시)
			- <u>내용</u>		(내용에 밑줄)
			- <h1>제목</h1>		(제목 표시, h1~h6까지 중요도 순으로 제목을 표시함)
			- <br>			(태그의 위치를 기준으로 한 줄 줄바꿈을 함)
			- <ul></ul>		(순서가 없는 리스트의 부모 태그)
			- <ol></ol>		(순서가 있는 리스트의 부모 태그)
			- <li></li>		(리스트 한 줄을 표현)
			- <title></title>		(페이지의 제목 표시)
			- <meta charset="~">	(페이지의 문자 표현 방식 결정)
			- <a></a>		(다른 페이지의 링크를 표현하는 태그)

		9) 태그를 사용할 때에는 해당 의미를 갖고 있는 태그를 사용하는 것이 더 좋음
		(ex : 단락 표시 시 줄바꿈 태그인 br의 사용보다 단락을 의미하는 p 태그의 사용이 더 좋음)
		-> 의미가 담긴 태그들을 사용할 시 검색 엔진이 컨텐츠를 찾을 때 도움을 줄 수 있음, 스크린리더기로 페이지를
		읽을 시 의미가 담긴 태그들이 의미 전달에 효율적

		10) 속성은 태그의 부연 설명
		-> img 태그 자체로는 이미지 표현에 대한 정보가 부족하지만, 부연 설명인 img 태그의 속성 중
		src 속성값에 표현할 이미지 경로값을 넣어 주면 이미지 표현이 가능해진다.

		11) 페이지의 구조를 표현하는 태그
		<html> : head와 body 태그, 즉 페이지 전체를 감싸는 태그
		<head> : 본문의 정보 표현 방식에 대해 설명하는 태그들을 감싸는 태그, 즉 메타 데이터 태그들의 부모 태그
		<body> : 본문의 내용을 표현하는 태그들의 부모 태그
		<!doctype html> : 해당 페이지가 html로 만들어졌다는 것을 명시하는 태그

		12) 웹 서버 <-> 웹 클라이언트
		웹 클라이언트에서 서버 측에 정보 요청(ex : 어떤 홈페이지 검색 시 해당 페이지의 웹 서버 주소에 홈페이지에 대한 내용 요청)
		서버에서는 요청받은 페이지를 찾은 후 응답함
