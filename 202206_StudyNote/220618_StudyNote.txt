1. 파이썬 시험 대비 개념들
	- 튜플 개념
		> 읽기 전용 리스트
		> 소괄호 ( ) 로 표현하며 생성할 때만 원소 생성 가능, 이후에는 원소를 수정할 수 없음

	- 튜플에서 사용 가능한 함수(★)
		> len, index, count 함수(각각 길이값, 위치값, 해당 원소 개수값 반환하는 함수)

	- 함수 개념
		> 특정한 기능을 수행하는 독립 모듈
		> 프로그램을 작성할 때 반복적으로 사용되는 코드를 미리 작성해 놓고 필요할 때 호출하여 사용하는 코드

	- 함수 장점?(★)
		> 코드를 함수화했을 시 코드 수정이 용이하다.
		> 기능 단위로 함수를 만들었기 때문에 코드의 간소화가 이뤄진다.
		> 자주 사용되는 코드들을 기능별로 묶어 함수로 만들어 두면 필요할 때 호출하여 사용할 수 있기 때문에 편리하다.

	- 지역변수/전역변수(★)
		> 전체 프로그램에서 인식되는 변수를 전역변수, 함수 안에서만 인식되는 변수를 지역변수라 부른다.
		> 함수 안에서 선언한 변수는 지역변수이고, 함수 밖에서 선언한 변수는 전역변수이다.
		> 지역/전역변수의 이름이 동일할 경우 지역변수가 우선된다.

	- 파일 읽기/쓰기의 3단계
		1) 파일 열기(open( ) 함수)
		2) 파일 읽기 혹은 쓰기(readline( ), readlines( ), read( ), writelines( ) 등 함수 활용)
		3) 파일 닫기(close( ) 함수)
		> 보통 파일을 열고 닫는 코드를 쓸 때 오류 발생에 대한 예외 처리를 위해 try~except 코드를 쓰는 경우가 많다.

	- 파일 열기
		> 파일을 읽을 경우
		변수명 = open("파일경로", "r", encoding="UTF-8")
		(파일을 읽을 때 encoding을 쓰지 않으면 ANSI로 오픈되며, encoding으로 UTF-8 지정 시 UTF-8 유니코드로 오픈된다.)
		> 파일을 쓸 경우
		변수명 = open("파일경로", "w")
		> 기존 파일 내용 뒤에 이어서 쓸 경우
		변수명 = open("파일경로", "a")

	* 경로에는 절대경로, 상대경로 모두 쓰일 수 있음

	- 파일 읽기
		> 파일변수명.readline( ) 함수
		파일을 한 라인씩 읽어서 변수에 가져온다.
		> 파일변수명.readlines( ) 함수
		파일 내용 전체를 가져와 리스트 형태로 저장한다.(단, 이 때 줄넘김 문자(\n)도 같이 가져오기에 이에 대한 처리도 해 줘야 한다.)
		> 파일변수명.read( ) 함수
		readlines 함수와 동일하게 파일 내용 전체를 가져오는데, 리스트 형태로 반환하는 것이 아니라 문자열 형태로 파일 내용
		전체 내용을 반환한다.
		

	- 파일 쓰기
		> 파일변수명.writelines( ~ ) 함수
		파일에 writelines 함수의 매개변수값을 작성한다.

	- 파일 닫기
		> 파일변수명.close( ) 함수
		파일을 닫는다.

	8) 규칙에 맞게 비밀번호를 작성하도록 하는 함수 작성하기
	(8글자 이상, 영문자 및 숫자로만 생성해야 하면 기호는 불가능)

	9) output.txt라는 파일에 문자열을 작성하고, 파일 읽기로 output 파일의 내용을 읽어오는 예제 만들기
